## Goal
Identify multibagger candidates (3–10x / 2–4yr) in upper-SMALL/MID Indian equities via a **SEPARATE** 3-stage hurdle/filter funnel (`multibagger_scores`, `/multibagger`), kept **OUT of `daily_picks`** (different objective + horizon).

- 🚫 **Phase 3 / Phase 4 — NOT pursued (deliberate).** The validation says the ranking edge is structurally zero-to-negative; more factors (Phase 3) or a cockpit/pipeline surface (Phase 4) would polish a tool that didn't clear its bar. Revisit only if a future regime or data source materially lifts capture.
- ✅ **Phase 5 — holding / conviction monitor (2026-06-04, [ADR 0040](../decisions/0040-multibagger-holding-not-selection.md)).** Reframe: multibagger is a HOLDING problem, not selection. Sector momentum validated 1–6mo (t+3.0–4.3, `tools/sector_signal_lab.py`; ensemble with the orthogonal macro engine t+3.0) but dead by 2–4yr → reused as the MONITORING signal, not a selection one. `tools/multibagger_monitor.py`: split-adjusted path study (81% of 3x+ winners draw ≥30% en route → stops eject winners, −30% stop 1.88→1.56x; winners −41%/10mo vs losers −62%/27mo underwater) + 3-policy backtest (conviction retains 94–100% of winners ≈ buy-hold, crushes naive stops). Per-name HOLD/WATCH/REVIEW (drawdown + sector momentum + relative strength) wired to `/multibagger` (`cockpit/api.py:_conviction_verdicts`). Basket sizing: 20–30 equal-weight names across sectors → P(≥1.5x) 85–91%. Plus 3 monthly PIT accumulators (`sector_analyst_breadth_pit`/`sector_sentiment_breadth_pit`/`sector_policy_pit`) started so the remaining at-entry sector signals are backtestable in ~12mo. **Bear stress test (2026-06-04, `--sector-stress`):** replaying the live rule on 15yr of NSE sector indices showed the un-guarded eject is INVERTED (fires at 2008/COVID capitulations that then doubled) → added a **market-regime guard** (suppress REVIEW when NIFTY SMALLCAP 250 ≥20% off peak; eject only idiosyncratic weakness) to `_conviction_verdicts`; removes the market-wide false ejects, keeps Realty-2011. Residual: idiosyncratic-laggard-in-recovery still mis-flags → REVIEW reframed as "reassess, not sell" (detailed in-app usage guide added to `/multibagger`). See [ADR 0040](../decisions/0040-multibagger-holding-not-selection.md).

## Done when
The funnel, gated on regime, shows a positive survivorship-corrected top-decile ≥3x capture across **≥3 independent regime windows**. **NOT met (2026-06-04)** — capture is 1–5% and top-decile edge is zero-to-negative in all 3 regimes. Stays validation-stage, NOT wired to production scoring; surfaced only as a junk-stripped **watchlist** with an honest regime disclaimer. Never calibrate thresholds to a single window (the known regimes calibrate to opposite extremes).
