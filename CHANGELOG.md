## 2026-05-10

- **F-track data layer landed.** Plan 0005 Phase A is in production for the data we control. `fundamentals_screener` now holds **681,256 rows × 2,119 stocks (86.6% of universe)** from Screener Premium's xlsx export, end-to-end via [sources/screener_pull.py](sources/screener_pull.py). LARGE 100/100, MID 150/150, SMALL 1,869/2,198. 329 fetch failures, all in SMALL — delisted / newly-listed / non-standard Excel templates. Logged in `screener_pull_errors`. Long-format schema (PK `(sid, period_end, period_type, line_item)`) absorbs new line items without DDL — see [ADR 0011](docs/decisions/0011-long-format-for-new-fundamentals-tables.md).
- **Schedules scraper (F1.2) — Trade Payables and 22 other broken-out line items.** Screener's xlsx rolls "Other Liabilities", "Borrowings", "Other Assets", "Fixed Assets" up to single buckets each, but the company page expands them via an undocumented JSON endpoint at `/api/company/{cid}/schedules/?parent=<row>&section=<section>&consolidated`. [sources/screener_schedules.py](sources/screener_schedules.py) hits that endpoint and writes into the same `fundamentals_screener` long-format table. RELI smoke test: 276 rows incl. `Trade Payables` (12 fiscal years), `Long term Borrowings`, `Short term Borrowings`, `Lease Liabilities`, `Plant Machinery`, `Land`, `Buildings`, `Trade receivables`, `Inventories`, `Loans n Advances`. **Settles plan-0005 question 2:** Trade Payables IS available; CCC is unblocked.
- **Two new factors shipped — F-track Phase B begins.** [signals/roic.py](signals/roic.py) — Return on Invested Capital, NOPAT = (PBT + Interest) × (1 − Tax/PBT), IC = Equity + Reserves + Borrowings, 3-yr median to suppress one-time gains, IC ≥ ₹50 cr floor to drop shell-sized stocks. 1,501 stocks scored; LARGE top-5 are NESTLE 65% / TCS 47% / HZNC 46% / BRITANNIA 41% / MARICO 38% (textbook quality compounders). [signals/fcf_yield.py](signals/fcf_yield.py) — FCF = OCF − Capex (Capex ≈ Δ(Net Block + CWIP) + Depreciation), yield = 3-yr median FCF / current market cap. 1,195 stocks scored; INFY 5.0%, TCS 4.4%, BRITANNIA 1.6%, RELI 0.4% (heavy capex). Both wired into pipeline.py as `signal_roic` and `signal_fcf_yield`. **Neither in scoring weights yet** — needs forward-return validation. **Gotcha discovered:** `stocks.market_cap_cr` is misnamed; values are stored in raw rupees, not crores. RELI shows 1.83×10^13 (= ₹18.3L cr in rupees). [signals/fcf_yield.py:39-46](signals/fcf_yield.py#L39-L46) divides by 1e7 with a comment.
- **Financial-sector exclusion was silently disabled — fixed.** `SCREEN["financial_sectors"] = ["Financial Services"]` but `stocks.sector` actually has `"Financials"`. The mismatch made the filter a no-op across `signals/roic.py`, `signals/accruals.py`, `signals/piotroski.py`, `signals/forensic.py`, and `tools/reconstruct_pit.py` — meaning **248 financial stocks have been silently scored by the main screener since v2 began**, contradicting the CLAUDE.md rule that financials must route through the financial sub-model. Fix is the one-character change `"Financial Services"` → `"Financials"` ([config.py:183](config.py#L183), commit `af94835`). Caught while sanity-checking ROIC on the LARGE tier — Indian banks at 3-5% ROIC dominated the bottom of the cross-section. Important caveat: this is **forward-only**. v1 archived snapshots already include financials baked into the IC samples, so historical t-stats from `tools/backtest_pit.py` are unchanged. Fully fix-applied historical backtest would require re-running `tools/reconstruct_pit.py` for all PIT dates (multi-hour compute, deferred).
- **`accruals.py` partial-composite-for-financials — fixed.** Same root cause but a separate symptom: even after the exclusion was repaired, accruals.py was emitting a non-null `accruals_signal` for 244 of 248 financials (because EPS-CV + beat-rate alone produced a composite when the accrual ratios came through as NaN). Fixed at [signals/accruals.py:283-285](signals/accruals.py#L283-L285).
- **Surveillance parser — fixed.** [sources/nselib_pull.py:549-595](sources/nselib_pull.py#L549-L595). NSE GSM API returns a bare list (not `{"data": [...]}`) with `gsmStage` + `survDesc` fields. nselib `dv.fno_security_in_ban_period()` returns a bare list (today empty, hence `df.empty` on a `list` raised `AttributeError`). Both crashed silently in the nightly cron for an unknown number of weeks. After fix: ASM/GSM/F&O write cleanly, including 3 new GSM rows from today.

## 2026-05-06

- **PIT-strict corporate-action adjustment.** v2's price adjustment is now PIT-clean — symmetric with the existing `knowable_quarterly` / `knowable_annual` / `knowable_shareholding` discipline for fundamentals. New table `corporate_adjustments` (PK `(sid, ex_date)`, columns `factor`, `n_events`, `inds`, `subjects`) holds 3,036 events: 92 SPLIT-only + 90 BONUS-only + 2,840 DIVIDEND-only + 17 multi-event days (same-day combos). New helper `tools.reconstruct_pit.apply_pit_adjustments(prices_pit, adjustments, eval_date)` composes only events with `ex_date <= eval_date` via reverse-cumprod + searchsorted (vectorized per sid). [tools/reconstruct_pit.py:283-324](tools/reconstruct_pit.py#L283-L324). `pit_momentum`, `pit_position_52w`, `pit_macd_bullish` switched from raw close to `adj_close`. `pit_fwd_return_20d` left on raw close (it's a backtest-realized-return metric, not a signal input). **Diagnostic:** 12-date apples-to-apples — raw close 0.745 → PIT-adj 0.862 mean Pearson vs v1 (+0.117 lift); essentially identical to the forward-adjusted-splits-only path from earlier today (+0.119) — confirms leakage in v1 yfinance is small in practice; the win is structural correctness, not numerical.
- **Deprecated/removed.** [tools/compute_splits.py] and [tools/apply_splits.py] deleted (superseded by `compute_corporate_adjustments.py` + the snapshot-aware helper). `stock_prices.adj_close` column left in place (1.3M rows of dead weight; safe to leave, drop in a future migration). `split_adjustments` table left in place (orphaned but inert).
- **Dividend parser added.** [tools/compute_corporate_adjustments.py](tools/compute_corporate_adjustments.py) parses 2,840 of 2,967 DIVIDEND events from `corporate_actions` (skipped: 2 with no parseable amount, 119 where pre-ex-date close wasn't in `stock_prices`, 6 where amount >= close_pre). Same `Re/Rs` handling as the splits regex. Dividend factor = `(close_pre - amount)/close_pre`. Same-day events (e.g. SPLIT+DIVIDEND on 2025-06-16) multiplied into a single combined factor — solves the same-day PK collision the old `split_adjustments` table had.
- **Split-adjustment parser at 100% coverage.** Earlier today: extended the splits regex to handle the canonical NSE `Face Value Split (Sub-Division) - From Rs X/- Per Share To Rs/Re Y/- Per Share` format and the `Re` (singular Rupee) marker. Parse rate **104/207 → 207/207**. (Now superseded by [tools/compute_corporate_adjustments.py](tools/compute_corporate_adjustments.py).)
- **Smart-beta indices: 13 indices populated** (was 9). Fixed three invalid names in `SMART_BETA_INDICES` ([sources/nselib_pull.py:432](sources/nselib_pull.py#L432)) — the names below match NSE's index master exactly: `NIFTY100 LOW VOLATILITY 30`, `NIFTY ALPHA LOW-VOLATILITY 30`. Added `NIFTY200 QUALITY 30` and `NIFTY100 EQUAL WEIGHT`. All 4 backfilled at 743 rows each (2023-05-08 → 2026-05-06). **Gotcha discovered:** nselib's `cm.index_data` silently caps wide-range queries at ~70 trading days regardless of `from_date`/`to_date`. Always paginate via `_months_back(N)` (monthly chunks).
- **upsert_df regression test landed.** [tests/test_smoke.py](tests/test_smoke.py) `test_upsert_df_preserves_untouched_columns` locks the contract that a partial-column upsert must not null untouched columns sharing the same PK row. Uses in-memory SQLite. All 7 smoke tests pass.

## 2026-05-02

- **`/flow` page rebuilt with a hand-rolled mini-DAG.** Four phase columns (Sources / Signals / Scoring / Output) side-by-side with `→` arrows between them, color-coded step pills (SUCCESS / FAILED / RUNNING / ABORTED / NEVER RUN), critical-fail steps get a thick red border. Sits above the existing detail-card grid in [cockpit/templates/flow.html](cockpit/templates/flow.html). No graph library — pure CSS grid + flexbox.
- **In-UI step rerun.** `↻` button on every step pill (mini-DAG) and every detail card → POST `/api/pipeline/rerun/{step_name}` → spawns `python pipeline.py --step <name>` as a detached subprocess via `start_new_session=True`. Logs to `output/rerun.log`. Backed by `rerun_step()` in [cockpit/api.py](cockpit/api.py) and a new POST route in [cockpit/app.py](cockpit/app.py). See [docs/decisions/0008-cockpit-write-surface.md](docs/decisions/0008-cockpit-write-surface.md) for the architectural shift (cockpit is now a write-side surface) and the five guardrails every future write endpoint must satisfy.
- **Duplicate protection on rerun.** Refuses with HTTP 409 `{"ok": false, "error": "<step> is already RUNNING"}` if a `pipeline_log` row for that step is RUNNING and younger than 5 minutes. Stale RUNNING (older) is treated as crashed and the rerun proceeds.
- **Auto-refresh on `/flow`** every 60s with a visible countdown; rerun click shortens the countdown to 8s so the status flip is visible without a manual reload.
- **Smoke tests updated.** Replaced the intermediate Mermaid-renderer tests with `test_flow_overview_returns_layers_and_failures` and `test_rerun_step_rejects_unknown_step`. All 6 pass via `python -m tests.test_smoke`.
- **Mermaid tried and rejected.** Earlier in the session the DAG was rendered via Mermaid (`cockpit/pipeline_dag.py`); its auto-layout collapsed 26 nodes into an unreadable horizontal stripe. The hand-rolled CSS-grid version replaced it; `cockpit/pipeline_dag.py` is deleted.
- **Dagster considered and rejected.** Conversation started as "should we add Dagster for pipeline visualization and asset-centric observability?" Per [docs/decisions/0002-no-prefect.md](docs/decisions/0002-no-prefect.md) (which explicitly named Dagster as an alternative and rejected it 23 days ago), and given no concrete observability gap that `SELECT * FROM pipeline_log` can't answer, the decision held. The narrower thing that *did* get accepted — cockpit may now mutate state via subprocess spawning — is recorded in ADR 0008.
